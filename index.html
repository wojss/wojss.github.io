Supervisorctl localhost:9001 无法连接

发表于 2016-04-06   |  

问题

在使用supervisorctl的时候，出现下面一个问题：

$ supervisorctl
http://localhost:9001 refused connection

原因

如果启动supervisord使用自定义路径的配置文件，比如：

sudo supervisord -c /mypath/supervisord.conf

supervisord和supervisorctl默认按顺序查找，使用第一个被查找到的配置文件。官方文档

  1. $CWD/supervisord.conf
  2. $CWD/etc/supervisord.conf
  3. /etc/supervisord.conf
  4. ../etc/supervisord.conf (相对路径)
  5. ../supervisord.conf (相对路径)

解决

在使用supervisorctl的时候应该加上相同的参数，比如：

sudo supervisorctl -c /mypath/supervisord.conf

或者创建一个软连接，这样可以省去后面的参数：

ln -s /mypath/supervisord.conf /etc/supervisord.conf

Bash快捷键

发表于 2016-03-10   |  

移动

Ctrl + a 行首，Home键相同
Ctrl + e 行末，End键相同
Ctrl + p 上一条命令，↑键相同
Ctrl + n 下一条命令，↓键相同
Ctrl + b 光标向左移，←键相同
Ctrl + f 光标向右移，→键相同
Alt + b 光标向左跳一个词，光标位于词首，Ctrl + ←键相同
Alt + f 光标向右跳一个词，光标位于次末，Ctrl + ←键相同
Ctrl + xx 在行首/行末间跳转

编辑

Ctrl + l 清屏，clean命令相同
Ctrl + d 删除，Delete键相同
Ctrl + h 退格，Backspace键相同
Alt + d 从光标向右删除一个词
Ctrl + w 从光标向左删除一个词
Ctrl + u 从光标删除到行首
Ctrl + k 从光标删除到行末
Alt + t 光标前后的词位置交换
Ctrl + t 光标前后的字符位置交换
Ctrl + y 粘贴最后一次切割的内容
Alt + r 撤销改变并恢复到原历史记录一样
Tab 补全文件/目录的名字

历史

Ctrl + r 在历史记录中搜索最后一条匹配的命令
Ctrl + o 直接执行通过Ctrl + r搜索到的命令
Ctrl + g 退出搜索模式
!! 执行上一条命令
!abc 执行最后以abc开头的命令
!abc:p 打印最后以abc开头的命令
!$ 上一条命令最后一个参数
Alt + . 同!$
!* 上一条命令所有参数
